# LIBRA World Manager v3.0.0

RisuAI용 장기 컨텍스트 관리 플러그인입니다.

LIBRA는 메모리, 엔티티, 관계, 세계관, 내러티브, 세션 전환, 로어북 동적 참조, 감정 분석, 스토리 작가, 감독 지시를 하나의 흐름으로 묶어 관리합니다.

## 핵심 기능

- 메모리 저장 / 검색 / GC / 롤백 감지
- 엔티티 / 관계 추출 및 추적
- 세계관 트리 및 규칙 관리
- 내러티브 추적 및 storyline 요약
- 스토리 작가 / 감독 프롬프트 계층
- 과거 대화 분석 / 과거 대화 재분석
- Hypa V3 지식 가져오기 및 구조화 반영
- 다음 세션으로 대화 이어가기
- 일반 로어북 동적 참조(RAG)
- 감정 분석
- CBS / GigaTrans / Lightboard 호환

## 현재 구조

- `lmai_memory`: 일반 기억
- `lmai_entity`: 인물/엔티티 정보
- `lmai_relation`: 관계 정보
- `lmai_world_graph`: 세계관 트리
- `lmai_narrative`: 진행 중인 줄거리 정보
- `lmai_story_author`: 스토리 작가 상태
- `lmai_char_states`: 캐릭터 상태 추적
- `lmai_world_states`: 세계 상태 추적

## 작동 방식 요약

### 1. 메모리와 구조 데이터

매 턴마다 대화에서 기억 후보를 추출하고, 중요도·감정·최근성·유사도를 기반으로 관리합니다.
메모리와 별도로 엔티티, 관계, 세계관, 내러티브는 `lmai_` 로어북 엔트리로 구조화되어 저장됩니다.

### 2. 로어북 동적 참조

캐릭터 로어북과 채팅 로컬 로어북을 합친 `effectiveLore`에서 일반 lore를 골라 현재 문맥과 관련 있는 항목만 참고 설정으로 주입합니다.

- `lmai_` 내부 엔트리는 내부 시스템이 직접 사용
- 일반 로어북은 동적 참조용 후보로 사용

### 3. 스토리 작가와 감독

스토리 작가는 다음 비트, 전개 방향, 장면 진행 힌트를 만듭니다.
감독은 그 위에서 더 강한 제약과 우선순위를 정리해 최종 프롬프트 지시를 만듭니다.

현재는 같은 턴에 병렬로 만들지 않고:

1. 스토리 작가 갱신
2. 감독 갱신

순서로 실행되어 감독이 최신 작가 상태를 읽습니다.

### 4. 세션 전환

`다음 세션으로 대화 이어가기`를 사용하면 새 채팅방을 만들고 다음 정보를 계승합니다.

- 메모리
- 엔티티 / 관계
- 세계관
- 내러티브
- 스토리 작가 상태
- 감독 상태
- 캐릭터 상태 / 세계 상태
- 직전 장면 요약

기존 대화 로그를 복사하는 것이 아니라, 상태와 기억을 계승한 새 세션을 만듭니다.

## 과거 대화 분석 / 재분석

### 과거 대화 분석

과거 메시지를 읽어 초기 구조 데이터를 만듭니다.

흐름:

1. 분석 가능한 메시지 선별
2. 메시지를 청크로 분할
3. `aux` LLM이 각 청크를 요약
4. `aux` LLM이 합성
5. 결과를 `lmai_` 구조 데이터로 반영

### 과거 대화 재분석

이미 있는 구조 데이터를 원문 대화 기준으로 다시 검토해 재구축합니다.

현재 재분석 파이프라인은 다음과 같습니다.

1. `aux` LLM이 새 `candidateData` 생성
2. 원문 대화를 여러 review window로 나눠 `aux` LLM이 부분 검토
3. `primary` LLM이
   - 현재 구조 데이터
   - 새 후보 데이터
   - 부분 review 결과
   를 함께 보고 최종 데이터를 판정
4. 최종 결과로 `lmai_` 구조를 재구축

즉, 생성과 최종 판정이 분리된 듀얼 LLM 구조입니다.

## Hypa V3 가져오기

Hypa V3 지식은 두 단계로 처리됩니다.

1. 원본 텍스트를 채팅 로어북에 `hypa_v3_import`로 저장
2. 그 텍스트를 구조화 분석해 `lmai_` 엔티티/관계/세계관 쪽에 병합

원본 Hypa lore는 보존되고, 구조화된 LIBRA 내부 데이터도 함께 생성됩니다.

## 듀얼 LLM

보조 LLM(`aux LLM`)을 활성화하면 다음 작업에 우선 사용됩니다.

- 과거 대화 분석 청크 요약
- 과거 대화 분석 합성
- 과거 대화 재분석 부분 review
- 스토리 작가 / 감독 / 보조 유지관리 작업

메인 LLM(`primary LLM`)은 최종 응답 생성과, 현재 기준으로 재분석 최종 판정에 사용됩니다.

## 권장 사용 흐름

1. 메인 LLM 설정
2. 필요하면 보조 LLM 설정
3. 필요하면 임베딩 설정
4. 메모리 프리셋 선택
5. 기존 채팅이면 과거 대화 분석 또는 재분석 실행
6. 필요하면 Hypa V3 가져오기
7. 스토리 작가 / 감독 모드 조정
8. 긴 서사는 세션 전환으로 이어가기

## 추천 세팅 예시

### 가벼운 기본 구성

- 메인 LLM: `gpt-4o-mini`급
- 보조 LLM: 비활성 또는 저비용 모델
- 임베딩: `text-embedding-3-small`
- 메모리 프리셋: `general`
- 스토리 작가: `비활성` 또는 `주도형`
- 감독: `비활성` 또는 `standard`

### 장기 RP / 서사형 구성

- 메인 LLM: 상위 GPT / Claude Sonnet / 강한 Gemini 계열
- 보조 LLM: 빠른 중급 모델
- 임베딩: OpenAI 또는 VoyageAI
- 메모리 프리셋: `sim_medium` 또는 `sim_large`
- 과거 대화 분석 범위: `부분(300)` 또는 `전체`
- 스토리 작가: `주도형`
- 감독: `strong`

### 보수적 보조 메모리 구성

- 스토리 작가: `비활성`
- 감독: `비활성`
- 로어북 동적 참조: 필요 시만 `ON`
- 감정 분석: `ON`
- 메모리 프리셋: `general`
