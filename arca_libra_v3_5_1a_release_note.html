Arca Release Note

LIBRA_World_manager V3.5.1a.js

V3.5.0에서 V3.5.1a로 올라오며 반영된 회귀 수정, LightBoard 호환성 보강, 추론 모델 대응 변경점을 한 번에 볼 수 있게 정리한 안내문입니다.

Release Focus
회귀 수정과 공존성 복원
Core Theme
LightBoard 충돌 완화 + 추론 모델 대응
User Impact
로어북 보존, 잘림 감소, 안정성 강화
한 줄 요약
V3.5.1a는 외부 로어북 보존 회귀, Story Author RAG 축소 회귀, LightBoard / XNAI 오인식, 추론 모델 응답 잘림 같은 실사용 문제를 실제 사용 흐름 중심으로 보강한 안정화 업데이트입니다.

1. 로어북 보존 회귀 수정

  • setLorebook가 다시 전체 로어북을 통째로 교체하지 않고, lmai_가 아닌 외부 항목을 보존하도록 수정되었습니다.
  • NPC List 같은 다른 플러그인 로어북과 LIBRA 로어북이 같은 채팅에서 더 안전하게 공존합니다.
  • 메모리/엔티티/관계/세계관을 갱신할 때 외부 로어북이 함께 지워지는 위험을 줄였습니다.

2. Story Author 로어북 RAG 복원

  • Story Author가 참조하는 일반 로어북 스니펫 수가 다시 3개 → 8개로 복원되었습니다.
  • 설정/세계관/인물 메모가 많은 채팅에서 Story Author가 장면 방향을 더 넓게 참고할 수 있습니다.
  • 장기 플롯이나 누적 설정이 많은 환경에서 보조 기획 품질이 다시 회복되는 쪽에 가깝습니다.

3. LightBoard / XNAI 호환성 보강

  • beforeRequest는 최근 pending 패턴뿐 아니라 전체 메시지에서 --- End of the log --- + # Job Instruction 조합도 감지합니다.
  • beforeRequest와 afterRequest 사이에 _lbRequestInFlight 연동이 추가되어, LightBoard 응답에 LIBRA의 PendingTurn이 잘못 등록되는 상황을 줄였습니다.
  • recoverIfNeeded()는 polling 경로뿐 아니라 모든 경로에서 LightBoard 활성 시 복구를 미룹니다.
  • syncMemory()는 LightBoard 활동 중에는 삭제 동기화를 건너뛰고, 직후 첫 실행에서는 orphan tracker만 정리해 롤백 오인식을 줄입니다.
  • <lb-xnai>, <lb-xnai-editing>, lb-xnai-gen/ 같은 패턴 감지가 보강되어 이미지 편집/재생성 계열 요청과 일반 서사 턴 구분이 더 안정적입니다.
  • LightBoard active 상태에서는 로어 저장을 더 오래 대기하도록 조정되어, 백그라운드 저장과 setChat/removeChat 충돌 가능성을 낮췄습니다.

4. 추론 모델 대응 설정 추가

  • 메인 LLM / 보조 LLM 설정에 Max Completion Tokens 항목이 추가되었습니다.
  • OpenAI 호환 추론 모델은 이제 reasoning_effort 사용 시 별도의 max_completion_tokens를 더 크게 줄 수 있어, finish_reason=length로 답변이 잘리는 상황을 줄입니다.
  • Gemini / Vertex thinking 모델도 추론 시 더 큰 출력 상한을 쓸 수 있도록 맞춰졌습니다.
  • Reasoning Budget Tokens는 계속 사용자가 직접 조절할 수 있고, 이제 추론 토큰 예산과 출력 토큰 상한을 분리해서 다룰 수 있습니다.

5. 정리하면 달라진 체감

  • 다른 플러그인 로어북이 LIBRA 저장 때문에 사라질 가능성이 줄었습니다.
  • Story Author가 장기 설정을 더 잘 참고합니다.
  • LightBoard / XNAI 내부 턴을 일반 대화로 오인해 구조 데이터가 오염되는 상황이 줄었습니다.
  • 추론 모델을 켰을 때 응답이 토큰 상한에 먼저 막혀 중간에 끊기는 문제가 완화되었습니다.

정리

V3.5.1a는 새 기능을 크게 늘린 버전이라기보다, 이미 추가된 시스템들이 실제 환경에서 덜 부딪히게 만드는 안정화 패치에 가깝습니다. 특히 외부 로어북 공존, LightBoard 충돌 완화, 추론 모델 출력 잘림 대응 쪽 체감이 가장 큽니다.
